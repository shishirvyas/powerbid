BID - Operations Help

Project Startup and Shutdown Guide

This file describes how to run the BID project on Windows using the root scripts and common maintenance commands.

1. Prerequisites

  • Node.js 20+ and pnpm 9+ installed.
  • PowerShell execution policy allows local scripts.
  • Database URL configured in app-next/.env.local.
node -v
pnpm -v

2. One-time Install

pnpm install

If database schema updates are pending:

pnpm --dir app-next db:push

3. Start the Project

Use the root script from repository root:

.\start.ps1

Options:

.\start.ps1 -Port 3000
.\start.ps1 -Detached
.\start.ps1 -Seed
  • -Detached: starts in a new PowerShell window.
  • -Seed: starts app then triggers demo seed endpoint.

App URL: http://localhost:3000

4. Stop the Project

Use the root stop script (kills listener on configured port):

.\stop.ps1
.\stop.ps1 -Port 3000

5. Daily Operations

Type-check

pnpm --dir app-next typecheck

Run app from pnpm directly

pnpm --dir app-next dev

Build for production

pnpm --dir app-next build
pnpm --dir app-next start

6. Troubleshooting

  • If port is busy, run .\stop.ps1 and restart.
  • If DB errors appear, verify BID_DATABASE_URL or POWERBID_DATABASE_URL.
  • If scripts are blocked, run PowerShell as admin once and set execution policy for current user.
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

Use production secrets in environment variables and never commit real credentials.
